# 🧭 개발 계획 (Claude Squad 기반)

## 🎯 개요
Claude Squad 구조를 바탕으로 파이프라인을 3개로 분리하고, main을 포함한 총 4개 워크플로우로 병렬 개발을 진행한다.  
각 파이프라인은 Claude Code를 활용하며, main에서는 Codex를 활용해 코드 리뷰 및 로그 기록을 수행한다.

---

## 🧩 워크플로우 구성

| 워크플로우 | 담당 Agent | 주요 역할 |
|-------------|-------------|-----------|
| 정보 검색 파이프라인 | Claude Code | 유물 관련 정보 검색 및 MD 파일 정리 |
| 스크립트 작성 파이프라인 | Claude Code | MD 정보를 기반으로 오디오가이드 스크립트 생성 |
| 오디오 생성 파이프라인 | Claude Code | 스크립트를 음성 파일로 변환 (TTS API) |
| Main | Codex | 각 파이프라인 결과에 대한 코드 리뷰, 통합 로그 및 리포트 작성 |

---

## 🚧 개발 단계

### 1️⃣ Mock-up 개발 단계
- 각 파이프라인별 **Mock-up (테스트 파일)** 작성
- 모든 Mock-up을 main 브랜치에 푸시
- 하나의 Agent(Claude Squad)가 전체 Mock-up의 통일성을 점검 및 조정
- 통합된 Mock-up 기준으로 다시 각 파이프라인 브랜치에 머지

### 2️⃣ 파이프라인 개발 단계
- 통일된 Mock-up 구조를 기준으로 실제 파이프라인 로직 구현
- Claude Code 기반으로 각 파이프라인 독립 개발
- Codex Agent가 main에서 코드 리뷰 및 품질 로그 자동화

---

## 🧠 추가 메모
- 각 단계의 결과물은 `outputs/` 폴더에 동일한 형식으로 저장되어야 함  
- main은 주기적으로 모든 브랜치의 변경사항을 확인하고 merge 로그를 남김  
- 병렬 개발 중 Claude Squad 간 커뮤니케이션은 주제별 comment thread로 관리
