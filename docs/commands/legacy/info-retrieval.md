# 정보 검색 파이프라인 후속 명령

## 📌 배경
- `git status` 기준으로 `outputs/.DS_Store`가 트래킹 상태로 남아 있어 macOS 전용 메타 파일이 리포지토리에 포함되었습니다.
- path_sanitizer 연동 후에도 dry_run 샘플을 커밋으로 제공하려면 관리 전략이 필요합니다.

## ✅ 수행할 작업
1. **macOS 메타 파일 정리**
   - `git rm --cached outputs/.DS_Store`로 기존 트래킹을 해제하고, 필요 시 `find . -name '.DS_Store' -delete` 스크립트를 추가해 재발을 방지하세요.
   - `.gitignore`에 이미 `.DS_Store`가 포함되어 있으므로 제거 후에는 자동으로 무시됩니다.
2. **샘플 데이터 정비**
   - dry_run이 생성한 목업 파일이 실제 검색 결과와 혼동되지 않도록, 샘플 산출물을 `outputs/info/examples/` 등 별도 폴더로 이동하거나 버전 관리에서 제외하는 정책을 제안하세요.
   - 메인 파이프라인이 참조하는 기본 경로(`outputs/info/`)에는 실제 실행 결과 또는 통합 테스트용 최신 파일만 유지합니다.
3. **통합 연동 확인**
   - `info_markdown_path`가 생성한 경로를 스크립트·오디오 파이프라인에서 정상적으로 읽어오는지 dry_run 및 실제 모드로 확인하고 로그에 경로가 일치하는지 기록하세요.

## 🧪 검증 체크리스트
- `git status`에서 `.DS_Store`가 더 이상 표시되지 않는지 확인.
- dry_run 실행 후에도 `outputs/info/` 루트에 불필요한 샘플이 남지 않는지 확인.
- 동일 키워드로 정보→스크립트→오디오 파이프라인을 실행했을 때 파일 경로가 모두 공백 유지 규칙을 따르는지 검증.
